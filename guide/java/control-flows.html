Control Flows

If-Statements

If

An if-statement will only run the code inside of it if the set condition evaluates to true

It follows this syntax:

if (condition) {
  // Run some code
}

Let's take a look at some examples:

// Ex 1
boolean gameOver = true;
// The if-statement will run as gameOver does equal false
if (gameOver == true) {
  System.out.println("Game Over!");
}

// Ex 2
int num = 44;
// The if-statement will not run as num does equal 44 so 44 != 44
if (num != 44) {
  System.out.println("num is not 44");
}

Else

  • This is the ending of an if-statement
  • Every else statement MUST be with an if-statement, but every if-statement doesn't need to be with an else statement (basically "else" can't stand alone, but "if" can)
  • There can only be one else statement in every if-else block (you can't do if-else-else)
  • The else statement will only run if the condition in the if-statement was false (and consequently the if-statement did not execute)
  • The else statement MUST be the last statement in an if-else or if-elseif-else block
  • The syntax looks like this:
    if (condition) {
      // Code
    } else {
      // Some other code
    }

Let's look at some examples:

// Ex 1
boolean gameOver = false;
if (gameOver == true) {
// Not going to execute as gameOver is not true
System.out.println("Game Over :(");
} else {
// This will run as the if condition returned false
System.out.println("The game is still going!");
}

// Ex 2
double num = 3.01;
if (num == 3.01) {
// This will run as num does equal 3.01
System.out.println("num is 3.01");
} else {
// This will not run as the if condition returned true
System.out.println("num is not equal to 3.01");
}

Else If

  • This is the last part of the if, else, and now else-if trio
  • An else-if statement MUST be with an if-statement, but every if-statement doesn't need to be with an else-if statement (basically "else if" can't stand alone, but "if" can)
  • Additionally, every else-if statement doesn't need to be with an else statement, nor does every else statement need to be with an else-if statement
  • Every else-if needs to be after the if-statement, but before an else statement (if there is an else statement)
  • Unlike the else statement, the else if statement accepts a condition. Like this:
    if (condition) {
      // Code
    } else if (condition) {
      // Some other code
    }
  • These are secondary conditions that will be tested if the main if-statement evaluates to false
  • You can have unlimited else-if statements in an if-elseif or if-elseif-else block

Now... time for some examples:

// Ex 1
int num = 0;
if (num == 0) {
  // This will run as num is 0
  System.out.println("num is 0");
} else if (num == 1) {
  System.out.println("num is 1");
} else if (num == 2) {
  System.out.println("num is 2");
} else {
  System.out.println("num is not 0, 1, or 2");
}

// Ex 2
int num = 1;
if (num == 0) {
  // This will not run as num is not 0
  System.out.println("num is 0");
} else if (num == 1) {
  // This will run as num is 1
  System.out.println("num is 1");
} else if (num == 2) {
  System.out.println("num is 2");
} else {
  System.out.println("num is not 0, 1, or 2");
}

// Ex 3
int num = 10;
if (num == 0) {
  // This will not run as num is not 0
  System.out.println("num is 0");
} else if (num == 1) {
  // This will not run as num is not 1
  System.out.println("num is 1");
} else if (num == 2) {
  // This will not run as num is not 2
  System.out.println("num is 2");
} else {
  // This will run as all the previous conditions failed
  System.out.println("num is not 0, 1, or 2");
}

For Loops

For loops are, as the name suggests, loops that will continuously execute a block of code until their condition becomes false.
To use a for loop, you can use a syntax like this:

for (variable declaration; condition; modify variable) {
  // Code
}

Let's use this in some examples:
The following code will output 100 numbers to the console.
For loops can be deceiving, though this for loop looks like it prints 100 numbers from 1 - 100, it actually starts at 0 and prints 100 numbers up until 100. This means it will print 0, 1, 2, 3... 98, and 99. It will never print 100 as the condition is i < 100, and 100 is not less than 100.

for (int i = 0; i < 100; i++) {
  System.out.println(i);
}

These for loops would do as you (likely) thought the for loop above did:

for (int i = 1; i <= 100; i++) {
  System.out.println(i);
}
// or
for (int i = 1; i < 101; i++) {
  System.out.println(i);
}

These for loops will print 1, 2, 3... 99, 100 as they both start at 1 and end at 100 or before 101.
Additionally, as you can see, I am using the variable i as that is just the standard convention, but any variable name of type will work.

Array Traversal with For Loops

Using for loops, we can also print out the contents of an array.
Basic Idea:

  • Start at i = 0 (as arrays start at index 0)
  • Go until the array's length, but stop at one less than the array's length
  • Print every value using i as a changing index

Put these ideas together:

String[] fruits = {“Apple”, “Banana”, “Orange”, “Pineapple”, “Jackfruit”};
for (int i = 0; i < fruits.length; i++) {
  System.out.println(fruits[i]);
}

This for loop will go through the array, fruits, and print out the values at fruits[0], fruits[1], fruits[2], fruits[3], and fruits[4]. So it will print:
Apple
Banana
Orange
Pineapple
Jackfruit

Enhanced For Loops

As you may have noticed, normal for loops can be a bit bulky, especially if all you're trying to do is traverse an array. This is where Enhanced For Loops can be used (also called for-each loops).
Enhanced for loops have a syntax like this:

for (type name : arrName) {
  // code
}

Let's use this in a few examples... the equivalent normal for loop will also be shown.
Let's start off by revamping our previous example

// Ex 1
String[] fruits = {“Apple”, “Banana”, “Orange”, “Pineapple”, “Jackfruit”};
// New
for (String fruit : fruits) {
  System.out.println(fruit);
}
// Old
for (int i = 0; i < fruits.length; i++) {
  System.out.println(fruits[i]);
}

// Ex 2
int[] numbers = {1, 2, 3, 4};
// New
for (int num : numbers) {
  System.out.println(num);
}
// Old
for (int i = 0; i < numbers.length; i++) {
  System.out.println(numbers[i]);
}

While working with enhanced for loops, there is one VERY important thing to note, which is that the variable you declare (String fruit : fruits) is a copy of the value in fruits. This means that modifying fruit, will NOT change the corresponding value in fruits. To change values, you must use a normal for loop.
Let's see what this in action:

int[] numbers = {1, 2, 3, 4, 5};
// Won't work as expected
for (int number : numbers) {
  number++; // Trying to add one to each value in numbers[]
}

The value of numbers is still [1, 2, 3, 4, 5] even after that for loop runs as number is a copy of the value and not the same (number is not numbers[i]).
To do this properly, you must use a normal for loop, like this:

for (int i = 0; i < numbers.length; i++) {
  numbers[i]++;
}

This will do as you expected and change numbers to [2, 3, 4, 5, 6].
Still don't get it? Here's what a normal versus an enhanced for loop does:

// Normal
int start = 1; // Starting array
x++; // Changes the value as expected

// Enhanced
int start = 1; // Starting array
int copy = start; // Enhanced for loop makes a copy
copy++; // This only changes copy, but not the original start variable

While Loops

While loops are similar to for loops, except they can run indefinitely. As you saw previously, the number of times a for loop runs is defined when it is written. But when a while loop is written, it can be written to run once, twice, or infinitely (do not recommend).
What is the syntax for a while loop?

while (condition) {
  // Code
}

Like for loops, while loops will run until their condition is met (evaluates to false). This also means it is possible to create an infinitely running while loop by giving a condition of true, as true will never be false, so the loop will run forever.

// Runs forever
// WARNING: May create a lot of lag and even crash your computer
while (true) {
  // Code
}

Now let's see some while loops in action!

int num = 0;
while (num < 10) {
  System.out.println(num);
}

This loop looks like it will run 10 times printing the numbers 0, 1... 8, and 9. But beware! We never increment num, so num will always be less than 10, so this loop is actually infinite.
Let's fix that while loop!

int num = 0;
while (num < 10) {
  System.out.println(num);
  num++;
}

This will work as expected, outputting 0, 1... 8, and 9.
What is the for loop version of this?

for (int i = 0; i < 10; i++) {
  System.out.println(i);
}
