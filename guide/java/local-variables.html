Variables Part 2

Variable Types

Variables are perhaps the most important concept in any programming language. You can think of them as boxes that can hold stuff. Java happens to be a statically typed language, meaning that the boxes are also tagged with a label (type) about what can be inside it. Below are some of the types in Java:

int

  • Stands for integer (whole number)
  • Values range from -2,147,483,648 to +2,147,483,647
  • Takes up 4 bytes of data

double

  • Stands for double precision floating point number
  • In case you are wondering there is a single precision floating point number using the keyword 'float'
  • A "binary version" of scientific notation
  • Usually used to store decimal values, but there will be precision errors
  • Takes up 8 bytes of space

boolean

  • A true or false value
  • Takes up 1 byte of space

char

  • Stands for character
  • A singular ASCII value
  • Takes up 1 byte of space

Up until this point, all the types are known as primitives and directly hold their values. More often than not however, we would like to have more complicated data structures, and that's where objects come into play. The variable type below is a reference type, which means that it does not directly hold its value.
Notice how they start with a capital letter:

String

  • Multiple chars stringed together
  • This type ALWAYS starts with a capital "S"

Variable Declaration & Assignment

Declaring a variable (not giving it a value):

type name;
int num;
boolean alive;

Assigning a value to an already declared variable:

name = value;
num = 3;
alive = true;

Declaring and assigning to a variable:

type name = value;
int num = 5;
double num2 = 5.275;

More examples:

String name;
name = "John";
boolean gameOver = false;
gameOver = true;

Variable Scopes

Variables can only be referenced in the scopes they are declared in and its subscopes. Scopes are surrounded by curly braces. Here's an example:

int one = 1;
{
  int two = one + one; // Fine
}
int three = one + two; // Error because two can
only referenced inside the braces

Variable names within a single scope must be unique. However, you can have variables with the same name in different scopes. If one of the scopes is inside the other scope, the variable in the outer scope will be shadowed by the other variable for the duration of the inner scope.

int num = 1;
// num is 1
{
  int num = 2;
  // num is 2
}
// num is 1 again
