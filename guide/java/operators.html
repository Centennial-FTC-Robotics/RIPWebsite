Operators

Arithmetic Operators

These are the basic operators you see in a math class, such as...

  • + for addition
  • - for subtraction
  • * for multiplication
    • Ex. 2*2 = 2x2 = 4
  • / for division
  • % for modulus (the remainder of a division operation)
    • Ex. 3 % 3 = 0 as 3 / 3 = 1 (goes in perfectly)
    • Ex. 6 % 3 = 0 as 6 / 3 = 2 (goes in perfectly)
    • Ex. 5 % 3 = 2 as 3 goes into 5 once with a remainder of 2
    • Ex. 9 % 5 = 4 as 5 goes into 9 once with a remainder of 4

Assignment Operators

These operators are used to modify the value of a variable.
You've already seen one of these operators... to declare variables!

// Use “=” for assigning values to variables
type name = value;

Additionally, there are more assignment operators:

  • var += [number] which is the same as var = var + number
  • var -= [number] which is the same as var = var - number
  • var /= [number] which is the same as var = var / number
  • var *= [number] which is the same as var = var * number
  • var++ which is the same as var += 1 or var = var + 1
  • var-- which is the same as var -= 1 or var = var - 1

Comparison Operators

These operators are used to compare two values
These are the basic comparison operators:

== (is equal to?)

  • Ex. 5 == 4 is false as 5 does not equal 4
  • Ex. (2 + 1) == 3 is true as 2+1 = 3 and 3 equals 3
  • It is very important to note that since a String is a reference type, == will not work with it. To compare String values you must do this:
    var.equals(“expression”);
    • Ex.
      String name = “John”;
      System.out.println(name == “John”) // Outputs: false as Strings are reference types
      System.out.println(name.equals(“John”)) // Outputs: true

! (logical not)

  • This essentially flips a boolean value
  • Ex. !true is false as not true is false
  • Ex. !false is true as not false is true

!= (not equal to?)

  • This checks if two values are not equal to each other
  • Ex. 1 != 1 is false as 1 does equal 1
  • Ex. 2 != 3 is true as 2 does not equal 3
  • Ex. “H” != “H” is true as Strings are reference types and their values can not be directly compared like this, to do that look at the next example.
  • Ex.
    String name = “John”;
    boolean isNotEq = !name.equals(“John”); // Now holds false as “John” does equal “John”

|| (logical OR)

  • Same thing as an OR gate
  • Only one side of the OR statement needs to be true in order for it to return true.
  • Short-circuiting with OR: This also allows you to skip evaluating the right side of an OR statement if the left side evaluates to true as if one side is true, the whole thing is true (this is also what a computer does)
  • Ex. true || false is true as one side is true
  • Ex. false || false is false as neither side is true and the whole thing returns false
  • Ex. (5 + 4 == 9) || (5 + 4 == 8) since 5 + 4 does equal 9, we can use short-circuiting as evaluate the whole OR statement as true

&& (logical AND)

  • Same thing as an AND gate
  • Both sides of the AND statement need to be true in order for it to return true.
  • Short-circuiting with AND: You can skip evaluating the right side of an AND statement if the left side evaluates to false as if one side is false, the whole thing is false (this is also what a computer does)
  • Ex. true || false is false as one side is false
  • Ex. false || false is false as neither side is true and the whole thing returns false
  • Ex. (5 + 4 == 8) || (5 + 4 == 9) since 5 + 4 does not equal 8, we can use short-circuiting as evaluate the whole AND statement as false

String Concatenation

We know that the arithmetic operators would work as normal on integers and doubles. However, only one arithmetic operator works with Strings, the plus (+) operator. It does exactly as you would think, it combines two strings. Sadly, you can not subtract, multiply, or divide strings.
Take a look at the following example:

String firstName = “John”;
String lastName = “Smith”;
String fullName1 = firstName + lastName;
// Now holds the value “JohnSmith”

You can add multiple strings in one go, which can allow for the addition of spaces as needed. From the example above, you can see there needs to be a space after John, we can do that:

String fullName2 = firstName + “ ” + lastName;
// Now holds the value “John Smith”
